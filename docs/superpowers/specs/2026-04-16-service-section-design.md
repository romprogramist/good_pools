# Секция «Сервисное обслуживание» — Дизайн-спецификация

## Обзор

Новая секция на `index.html`, представляющая регулярное сервисное обслуживание бассейнов как дополнительную услугу после строительства. Содержит краткое описание, две метрики и форму-опросник для записи. Отправка заявки — через предзаполненное сообщение в WhatsApp (бэкенда у сайта нет).

## Размещение и навигация

- Новая секция `<section class="service" id="service">` на `index.html` между `WHY US` и `CONTACTS`.
- Пункт «Услуги» в мобильном оверлейном меню (`nav.menu-overlay`) получает `href="#service"` вместо текущего `#`.
- Секция доступна по якорю `#service` — ссылки с других страниц (`models.html`, `portfolio.html`) могут указывать на `index.html#service` при необходимости (вне текущего скоупа).

## Заголовок секции

Повторяет существующий паттерн (как `WHY US`, `PORTFOLIO`):

- **Лейбл:** «Сервис» — 12px, uppercase, letter-spacing 5px, цвет акцентный (#0ea5e9 или эквивалент из остальных секций).
- **Заголовок:** `<h2>` в двух строках — «**СЕРВИСНОЕ**» (bold 900) + «ОБСЛУЖИВАНИЕ» (thin 200), размер как у соседних секций.
- **Описание:** «Дополнительная услуга после строительства — регулярное обслуживание бассейна нашими специалистами».

## Лейаут

Двухколоночный flex, как `.why-us-inner`:

- **Слева (`.service-left`):** заголовок + описание + блок метрик.
- **Справа (`.service-right`):** карточка с формой (скругление 20px, как у контактной карточки/карты).
- Gap между колонками — 40–60px (подобрать под существующие секции).
- На мобильных (≤ 900px) — одна колонка, форма под левым блоком.

## Блок метрик

Две метрики в существующем стиле `.why-stat` (переиспользуются те же классы или добавляется `.service-stat` с идентичными стилями):

| Значение | Подпись |
|---|---|
| **80+** | клиентов на регулярном сервисе |
| **3** | специалиста ежедневно |

Отображаются в горизонтальном flex-ряду с gap 32px (как `.why-stats`). На мобильных — остаются в ряд (2 элемента умещаются), либо столбец при очень узких экранах.

## Форма-опросник

### Поля

| Поле | HTML-тип | Обязательное | Валидация |
|---|---|---|---|
| Ваше имя | `<input type="text">` | да | `required`, minlength 2 |
| Телефон | `<input type="tel">` | да | `required`, `pattern="[0-9+\\-\\s()]{10,}"` |
| Размер бассейна | `<input type="text" placeholder="6×3×1.5 м">` | да | `required` |
| Год постройки | `<input type="number" min="1990" max="2026">` | да | `required` |
| Автоматика дозирования химии | `<fieldset>` с тремя `<input type="radio">` | да | `required` |
| Комментарий | `<textarea rows="3">` | нет | — |

Варианты для «Автоматика дозирования химии»: «Да», «Нет», «Не знаю».

### Стили формы

- Поля: прозрачный/тёмный фон в духе сайта, тонкий border в палитре секции, скругление 12px, padding 12–14px 16px, font-size 14–15px.
- Лейблы над полями — 11–12px, uppercase, letter-spacing, приглушённый цвет (как `.why-stat-label`).
- Ошибки валидации: `aria-invalid="true"` + 1px красный border (#ef4444 или ближайший тон). Никаких стандартных всплывающих пузырей браузера — `novalidate` на форме, ручная валидация на submit.
- Radio-группа «Автоматика» — горизонтальные pill-кнопки (3 штуки), активная подсвечена акцентом.

### Кнопка отправки

- Одна основная кнопка: «**Отправить заявку в WhatsApp**» (или краткое «Записаться на сервис»).
- Стиль — существующий CTA сайта (та же высота, радиус, палитра, что у кнопок в hero / «Смотреть все работы»).
- Под кнопкой — текстовая ссылка-алтернатива: «Или напишите нам в Telegram →». Ведёт на `https://t.me/+79613201050` (без префила — ограничение Telegram для личных чатов).

### Логика отправки

- Клик по основной кнопке: JS валидирует поля → если ок, собирает текст сообщения → открывает `https://wa.me/79613201050?text=<encoded>` через `window.open(url, '_blank', 'noopener')`.
- Формат текста сообщения:
  ```
  Заявка на сервисное обслуживание

  Имя: <name>
  Телефон: <phone>
  Размер бассейна: <size>
  Год постройки: <year>
  Автоматика химии: <да/нет/не знаю>
  Комментарий: <comment или «—»>
  ```
- После успешного открытия WhatsApp — показать inline-подтверждение под кнопкой («Открываем WhatsApp…») на 3 сек.
- Если валидация не прошла — у первого невалидного поля ставится `aria-invalid`, фокус ставится на него, под полем показывается короткий текст ошибки.

## Файлы

- `index.html` — разметка новой секции, обновление `href` для пункта «Услуги» в меню, подключение `js/service.js`.
- `css/style.css` — блок стилей `.service`, `.service-inner`, `.service-left`, `.service-right`, `.service-form`, `.service-field`, `.service-radio`, `.service-submit`, `.service-tg-link`, а также адаптивные правила для ≤ 900px и ≤ 600px брейкпоинтов, используемых в остальном файле.
- `js/service.js` — новый файл: валидация формы, построение текста сообщения, открытие `wa.me`, отображение inline-статуса.

## Данные и константы

Номер WhatsApp/Telegram — `+79613201050` (уже используется в `.contact-messengers` и футере). Вынести в одну константу в `js/service.js` (`const CONTACT_PHONE = '79613201050';`).

## Доступность

- Все поля имеют связанные `<label for="...">`.
- Radio-группа обёрнута в `<fieldset>` с `<legend>`.
- Кнопка submit — нативная `<button type="submit">`.
- Сообщения об ошибках связаны с полями через `aria-describedby`.
- Фокус-стили форм видимы (не убираем outline без замены).

## Адаптивность

- **Desktop (> 900px):** две колонки, форма справа.
- **Tablet (601–900px):** одна колонка, метрики в горизонтальном ряду, форма ниже.
- **Mobile (≤ 600px):** одна колонка, поля формы на всю ширину, метрики в ряд (две штуки умещаются при `flex: 1`).

## Вне скоупа

- Многошаговый мастер-опросник.
- Капча / антиспам.
- Бэкенд для хранения заявок, CRM-интеграция.
- Отправка на e-mail.
- Личный кабинет клиента / история заявок.
- Отдельная страница `service.html` — при росте контента в будущем.
