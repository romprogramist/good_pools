# Страница «Все модели» — дизайн-спецификация

**Дата:** 2026-04-14
**Статус:** одобрено, готово к реализации
**Контекст:** демо-макет для показа заказчику, позже данные будут управляться через админ-панель

## Цель

Создать страницу `models.html`, доступную по клику на кнопку «Все модели» в секции «КАТАЛОГ ЧАШ» главной страницы. Страница показывает весь ассортимент (6 категорий, ~18 моделей) с клиентским фильтром по категориям. Тёмная тема для визуального контраста с остальным (светлым) сайтом.

## Scope

- Статический визуал-прототип
- Данные хардкодом в JS-массиве, без бэкенда
- Фильтр работает на клиенте (show/hide через `classList`)
- Шесть категорий × три модели = 18 карточек
- Навигация: клик «Все модели» на главной → `models.html`

## Вне scope (YAGNI)

- Поиск, сортировка, пагинация
- Модалка/отдельная страница конкретной модели
- Реальные фотографии (используются SVG-иконки категорий из `images/categories/`)
- Перекрас остального сайта в тёмную тему
- Бэкенд, API, админка

## Архитектура

### Файлы

| Файл | Роль |
|---|---|
| `models.html` | Разметка страницы. Копия шапки и мобильного меню из `index.html` + `<main class="models-page">` |
| `js/models.js` | Массив `MODELS` (18 объектов), функция `renderGrid()`, обработчик фильтра |
| `css/style.css` | Новая секция `/* === ALL MODELS PAGE === */`, все селекторы под префиксом `.models-page` |
| `index.html` | Кнопка `.btn-all-top` меняется с `<button>` на `<a href="models.html">` |
| `catalog.html` | Та же замена `<button>` → `<a>` (консистентно, файл живёт как standalone-прототип) |

`js/main.js` реюзается ради гамбургер-меню и Escape-закрытия.

### Компонент `.mcard`

Вертикальная карточка (в отличие от двухколоночной `.pcard` на главной), чтобы помещалось 3 в ряд.

```
┌─────────────────────┐
│   [SVG категории]   │  квадрат ~260×260, тёмный фон с cyan-свечением
│        badge        │  опционально: "Хит продаж" / "Новинка"
│              size   │  размеры в правом нижнем
├─────────────────────┤
│ ИМЯ МОДЕЛИ          │  крупно, светлый
│ Серия / категория   │  cyan, мелко, uppercase
│ 2-строчное описание │
│ 8.0 · 3.6 · 1.6 м   │  specs одной строкой
│ от 1 250 000 ₽      │
└─────────────────────┘
```

Hover: `translateY(-6px)` + cyan-свечение `box-shadow`. Клик — ничего не делает.

### Фильтр

- Чипы рендерятся из списка категорий
- Активный чип: фон `#0ea5e9`, белый текст
- Неактивный: прозрачный фон, граница `rgba(14,165,233,0.3)`, текст `#94a3b8`
- Клик: снимает `.active` со всех → ставит на нажатый → перебирает карточки → тогглит `.hidden` по сравнению `chip.dataset.category` vs `card.dataset.category`
- Чип «Все» показывает все карточки
- По умолчанию активен «Все»
- Без анимации repositioning — простой show/hide достаточно для демо

### Данные моделей (18 штук)

Структура объекта:

```js
{
  id: 'hiit',
  name: 'HIIT',
  category: 'composite',       // ключ категории
  series: 'Спортивная серия',
  desc: 'Короткое 1-2 строчное описание',
  image: 'images/categories/composite-pool.svg',
  sizes: { length: 8.0, width: 3.6, depth: 1.6, unit: 'м' },
  price: 'от 1 250 000 ₽',
  badge: 'Хит продаж'          // опционально
}
```

Для категорий без линейных размеров (мебель, надувные спа) `sizes` — объект с другими полями, которые рендер отобразит как «специальные характеристики».

**Композитные (composite):** HIIT, ZEN, TETTA
**Кастом (custom):** CUSTOM Classic, CUSTOM Infinity, CUSTOM Lagoon
**Джакузи-спа (jacuzzi):** SPA Duo, SPA Family, SPA Premium
**Надувные спа (inflatable):** INFLATE Round, INFLATE Square, INFLATE Jet
**Фурако (furako):** FURAKO Classic, FURAKO Oval, FURAKO XL
**Мебель (furniture):** LOUNGE Chair, LOUNGE Sofa, LOUNGE Bar

Цены, размеры и описания — правдоподобные плейсхолдеры.

accent cyan (active): #0ea5e9

Header (шапка сайта) на тёмной странице тоже получает тёмный фон — переопределяется через `.models-page ~ .header` или `body.dark-theme .header`. В спеке: добавить класс `body.models-dark` на `<body>` в `models.html`, и написать оверрайды для `.models-dark .header` в style.css.

### Hero страницы

H1 — те же `.bold`/`.thin` классы, что на главной (в `.hero-title-bold` / `.hero-title-thin`), но адаптированные к тёмной теме. Под H1 — строка `18 моделей · 6 категорий` в `#22d3ee`.

## Тестирование

- Открыть `http://localhost:8000/models.html` в браузере
- Проверить: грузятся 18 карточек, 6 чипов в фильтре
- Кликнуть каждый чип → проверить что остаются только карточки нужной категории
- Кликнуть «Все» → все возвращаются
- На мобильной ширине (DevTools, 375px) → 1 колонка, чипы переносятся
- С главной (`index.html`): клик «Все модели» → переход на `models.html`
- Hover на карточку → свечение
- Гамбургер-меню → открывается/закрывается

## Риски и решения

- **Тёмная шапка конфликтует со светлой на главной** → тёмные оверрайды срабатывают только при `body.models-dark`, светлая главная не трогается.
- **SVG-иконки одинаковые для всех моделей категории** → осознанный компромисс для демо, заказчик понимает что это плейсхолдеры.
- **18 карточек — много для одного экрана** → 3 колонки × ~6 строк, скролл приемлемый; фильтр позволяет быстро сузить список.
