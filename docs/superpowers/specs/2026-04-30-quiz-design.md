# Дизайн квиза «Рассчитать стоимость»

**Дата:** 2026-04-30
**Статус:** утверждён, готов к плану реализации
**Скоуп:** только фронтенд. Бэкенд (`/api/leads`, миграция БД, уведомления) — отдельная итерация.

## Цель

На главной странице `index.html` по клику на CTA-кнопку `РАССЧИТАТЬ СТОИМОСТЬ` (hero-секция) открывается модальный 7-шаговый квиз. Содержимое и логика — 1:1 с референсом `https://xn--80ablclk2abatqa7b6b2c.xn--p1ai/`. Визуальный стиль — под существующий сайт `good_pools`.

## Триггер и интеграция

- **Триггер квиза:** только `index.html:120` — кнопка `.hero-cta-btn`.
- Якорь `href="#service"` с этой кнопки убирается; меняется на `<button type="button" class="hero-cta-btn" data-quiz-open>РАССЧИТАТЬ СТОИМОСТЬ</button>`.
- Существующий блок `#service` с формой «Запись на сервис» (`#serviceForm`, `index.html:295–348`) **не трогаем**. Он остаётся доступным через ссылки `Оставить заявку` в меню (`index.html:66`) и в футере (`index.html:425`), а также через кнопку `Бесплатная консультация` (`index.html:148`, если у неё уже есть обработчик — мы его не меняем).
- Модальный `<dialog>` JS вставляет в `document.body` при инициализации; в HTML руками не пишем.

## State и поток

Единая переменная-объект на модуль:

```js
const state = {
  step: 1,                    // 1..7
  answers: {
    size: null,
    finish: null,             // 'composite' | 'concrete' | 'consult'
    filtration: null,         // 'yes' | 'consult'
    options: [],              // массив id выбранных опций шага 4
    budget: null,             // '1-2' | '2-3' | '3-4'
    timing: null,             // 'soon' | 'season' | 'browsing'
    name: '',
    phone: ''
  }
};
```

**Конфиг шагов** — массив длиной 7, единое место правды:

```js
const STEPS = [
  { id: 'size',       title: 'Размер бассейна',     type: 'single',   options: [...] },
  { id: 'finish',     title: 'Тип отделки',         type: 'single',   options: [...] },
  { id: 'filtration', title: 'Фильтрация воды',     type: 'single',   options: [...] },
  { id: 'options',    title: 'Дополнительные опции',type: 'multi',    options: [...] },
  { id: 'budget',     title: 'Бюджет',              type: 'single',   options: [...] },
  { id: 'timing',     title: 'Сроки',               type: 'single',   options: [...] },
  { id: 'contacts',   title: 'Контакты',            type: 'contacts'                  }
];
```

**Рендер:** функция `render()` берёт текущий `STEPS[state.step - 1]`, чистит `.quiz-body`, собирает разметку через шаблонные строки. Вызывается при каждом переключении шага.

**Навигация:**
- `← НАЗАД` → `state.step--`, `render()`. На шаге 1 кнопка скрыта.
- `ДАЛЕЕ →` → `validateStep(state.step, state.answers)`; если ok — `state.step++`, `render()`; иначе показать ошибку.
- На шаге 7 «Далее» → `ПОЛУЧИТЬ ПРЕДЛОЖЕНИЕ` → submit (см. ниже).
- Закрытие — крестик, `Esc`, клик по бэкдропу. `state` при закрытии сбрасывается (повторное открытие — с шага 1, поля пустые).

**Сохранение прогресса между сессиями:** не делаем. Ни `localStorage`, ни cookies.

## Submit (без бэкенда)

На шаге 7 после успешной валидации:

1. `console.log('[quiz] submit', state.answers)` — пока для отладки.
2. Явный комментарий-маркер в коде: `// TODO: replace with POST /api/leads when backend is ready`.
3. `.quiz-body` заменяется на финальный экран:
   - Заголовок: `Спасибо! Заявка принята`.
   - Подзаголовок: `Менеджер свяжется с вами в течение 10 минут` (формулировка с референса).
   - Кнопка `Закрыть` — закрывает диалог.
4. На финальном экране кнопок «Назад / Далее» нет.

## Валидация

| Шаг | Поле | Правило | Сообщение |
|---|---|---|---|
| 1 | `size` | выбран один | `Выберите размер` |
| 2 | `finish` | выбран один | `Выберите тип отделки` |
| 3 | `filtration` | выбран один | `Выберите вариант` |
| 4 | `options` | **не требуется**, можно идти с пустым массивом | — |
| 5 | `budget` | выбран один | `Выберите диапазон` |
| 6 | `timing` | выбран один | `Выберите срок` |
| 7 | `name` | `name.trim().length >= 2` | `Введите имя` |
| 7 | `phone` | очищенная от не-цифр строка имеет длину `>= 10` | `Введите телефон` |

- Ошибка показывается в `.quiz-error` под заголовком шага. Очищается при любом изменении выбора/инпута.
- Кнопка `ДАЛЕЕ →` визуально не дизейблится — клик при невалидном состоянии показывает ошибку (это понятнее, чем «мёртвая» кнопка).
- Маска телефона **не делаем** (paste/backspace/нерусские номера — отдельная история).
- Отдельного чекбокса согласия нет — текст под кнопкой submit (как на референсе).

**Системные ошибки:**
- `dialog.showModal()` бросает в очень старых браузерах → ловим в try/catch, fallback `alert('Ваш браузер не поддерживает эту форму, обновите его.')`. Для прода это редкий случай (нативный `<dialog>` поддержан везде с 2022).

## Чистые функции (тестируемые в изоляции)

- `validateStep(step, answers) → { ok: true } | { ok: false, message }`
- `normalizePhone(raw) → string` (только цифры)

Скрипт подключается как обычный `<script src="js/quiz.js" defer>` (не ES-модуль), поэтому `quiz.js` в самом конце вешает эти функции на `window.__quiz = { validateStep, normalizePhone }`. `quiz.test.html` подключает тот же `quiz.js` и обращается к ним через `window.__quiz`.

## Тестирование

**1. Юнит-тесты — `js/quiz.test.html`.**

HTML-страница без зависимостей: `import` из `quiz.js`, `<script>` с десятком `console.assert(...)` для `validateStep` и `normalizePhone`. Открыл в браузере → в консоли `OK: N/N` или красные fail. Когда понадобится «настоящий» сетап — мигрируем.

**2. Smoke-чеклист (ручной прогон в браузере перед коммитом):**

- [ ] Клик `РАССЧИТАТЬ СТОИМОСТЬ` → модалка открывается на шаге 1.
- [ ] Без выбора клик `ДАЛЕЕ` → ошибка «Выберите размер», шаг не сменился.
- [ ] Выбор → ошибка ушла → `ДАЛЕЕ` ведёт на шаг 2.
- [ ] `НАЗАД` с шага 2 возвращает выбор шага 1 (state сохранён в сессии).
- [ ] Шаг 4 — выбор нескольких опций работает (toggle).
- [ ] Шаг 4 — пустой выбор пропускает дальше без ошибки.
- [ ] Шаг 7 — невалидный телефон («abc») → ошибка.
- [ ] Submit с валидными полями → `console.log` с `state.answers`, экран благодарности.
- [ ] Кнопка «Закрыть» на финале и `Esc` закрывают модалку.
- [ ] Повторное открытие — с шага 1, поля пустые.
- [ ] Мобильная ширина (DevTools, 375 px) — модалка влазит, кнопки доступны.
- [ ] Существующая форма `#service` не сломана; ссылки `Оставить заявку` в меню и футере по-прежнему ведут на `#service`.

**Чего не тестируем в этой итерации:** end-to-end submit (нет бэка), визуальный регрешн, accessibility сверх того, что даёт нативный `<dialog>`.
