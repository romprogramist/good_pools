# Карточки товаров (бассейны) + фид для Яндекс.Директа

Дата: 2026-06-08

## Цель

Запустить товарную кампанию в Яндекс.Директе (конкурент — composit-pools.ru). Для этого сайту нужно:
1. На каждый бассейн — отдельную страницу-карточку на своём URL (посадочная под рекламу + SEO).
2. Фид (XML-файл с товарами) по URL, который специалист по Директу подключит к кампании.
3. Sitemap для индексации новых страниц в поиске.

Настройка самой кампании и бюджета («выставить максимальный бюджет») — зона специалиста по Директу, не входит в эту работу.

## Принципы / ограничения

- **Только существующие данные.** Карточки наполняются из текущей таблицы `models` и `model_images`. Новые поля и новый контент НЕ заводим.
- **Без миграций БД и без изменений админки.**
- Всё реализуется в уже-деплоящихся папках (`routes/`, `views/`, `js/`, `css/`, `lib/`) — список rsync в `.github/workflows/deploy.yml` менять не нужно.
- Домен: `хорошиебассейны.рф` (punycode `xn--80ablclk2abatqa7b6b2c.xn--p1ai`). В фиде/sitemap — абсолютные URL.

## Компоненты

### 1. Страница-карточка `GET /pool/:slug`

Сервер достаёт бассейн по `slug` из `models` (+ фото из `model_images`) и рендерит HTML в стиле сайта.

Блоки (всё из существующих данных, показываем только заполненное):
- Хлебные крошки: Главная → Каталог → {категория} → {название}
- Галерея фото (`model_images`, обложка первой)
- Заголовок + серия + бэйдж
- Таблица характеристик: длина / ширина / глубина (`length_m`, `width_m`, `depth_m`)
- Цена строкой как на сайте (поле `price`, напр. «от 1 250 000 ₽»)
- Описание (`description`)
- CTA «Оставить заявку» — текущая форма лидов (`POST /api/leads`), с указанием бассейна (source/page)
- Микроразметка `schema.org/Product` (JSON-LD): name, description, image, brand «Хорошие Бассейны», offers{price (число), priceCurrency RUB, availability InStock}

Несуществующий slug → 404 в стиле сайта.

### 2. Карточки на `models.html` ведут на `/pool/:slug`

Сейчас карточки в сетке никуда не ведут. Правка `js/models.js`: каждая карточка ссылается на `/pool/{slug}`.

### 3. Фид `GET /feed.xml`

Генерируется на лету из `models` (всегда актуален). Формат — **Google Shopping XML** (принимается Яндекс.Директом для «прочих типов»).

На каждый бассейн `<item>`:
- `g:id` — slug
- `g:title` — название (+ серия)
- `g:link` — `https://хорошиебассейны.рф/pool/{slug}`
- `g:image_link` — прямая ссылка на обложку из `uploads/` (большие, требование «мин. 500px» проходит)
- `g:description` — `description`
- `g:price` — число + ` RUB` (число парсится из строки `price`)
- `g:availability` — `in stock`
- `g:brand` — «Хорошие Бассейны»
- `g:product_type` — категория

**Бассейны без распознанной цены** (бетонные «По проекту») — страница-карточка есть, но в фид НЕ попадают (Яндексу нужна цена). Если позже проставить им «от»-цену, появятся в фиде автоматически.

### 4. Sitemap `GET /sitemap.xml` + `robots.txt`

- `/sitemap.xml` генерируется на лету: статические страницы (главная, models, portfolio и т.д.) + все `/pool/:slug`.
- `robots.txt` (сейчас отсутствует) со строкой `Sitemap: https://хорошиебассейны.рф/sitemap.xml`.

### Общий хелпер `lib/price.js`

Парсит строку цены → число (`«от 1 250 000 ₽»` → `1250000`; нет цифр → `null`). Один источник правды для schema.org-цены и фида.

## Файлы

- `routes/catalog.js` (новый) — маршруты `/pool/:slug`, `/feed.xml`, `/sitemap.xml`; подключить в `server.js`.
- `views/pool.ejs` (новый) — шаблон карточки.
- `lib/price.js` (новый) — парсер цены.
- `js/models.js` — ссылки карточек на `/pool/:slug`.
- `css/pool.css` (новый) — стили карточки.
- `robots.txt` (новый).

## Обработка ошибок

- `/pool/:slug` неизвестный slug → 404.
- `/feed.xml`, `/sitemap.xml` — всегда валидный XML; товары без цены пропускаются из фида.

## Проверка (вручную, тест-фреймворка нет)

- Открыть несколько `/pool/...`: данные, фото, цена, CTA, крошки.
- Заявка с карточки падает в лиды/почту/телеграм.
- `/feed.xml` — открыть, прогнать через валидатор фидов Яндекса.
- schema.org — валидатор разметки Яндекса.
- `/sitemap.xml` — валидный, содержит все карточки.

## Вне scope

- Настройка кампании/бюджета в Директе (специалист).
- Корзина/онлайн-оплата (CTA = заявка, как на остальном сайте).
- Цвета/материалы/доп.опции/гарантия/чертежи (нет данных — не заводим).
