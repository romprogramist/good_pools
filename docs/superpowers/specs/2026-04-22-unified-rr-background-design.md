# Единый фон сайта под секцию «От рендера до реальности»

## Цель

Сделать фон всех страниц сайта одинаковым — как в секции `.rr` (`#render-reality`) на главной. Сейчас каждая секция имеет свой оттенок тёмного (или ярко-синий, как `.why-us`), визуально страница распадается на «полосы».

## Эталон (откуда берём фон)

Из `css/style.css`, `.rr` (≈ строка 3418):

```css
background: linear-gradient(180deg, #0a1014 0%, #0f1419 60%, #0a1014 100%);
```

Декоративное голубое свечение (`.rr::before`) — **только в `.rr`, остальные секции не трогаем.**

## Подход

**Положить градиент на `body` с `background-attachment: fixed`** и снять `background` со всех секций, которые сейчас перекрывают тело своим тёмным цветом. `fixed` нужен, чтобы при скролле длинной страницы градиент не растягивался незаметно — он отрисовывается на размер вьюпорта и остаётся «как в `.rr`».

## Что меняем в `css/style.css`

### 1. `body` (строка ≈9)

```css
body {
  font-family: 'Montserrat', 'Segoe UI', sans-serif;
  background: linear-gradient(180deg, #0a1014 0%, #0f1419 60%, #0a1014 100%) fixed;
  overflow-x: hidden;
  -webkit-font-smoothing: antialiased;
}
```

### 2. Снять `background` с этих селекторов (фон будет проступать от `body`)

| Селектор | Было | Действие |
|---|---|---|
| `.hero` (≈274) | `#111` | удалить строку `background` |
| `.catalog` (≈521) | `#181c20` | удалить |
| `.why-us` (≈1131) | `linear-gradient(135deg, #0c4a6e ... #0284c7)` (ярко-синий) | удалить |
| `.showroom` (≈1302) | `#111` | удалить |
| `.service` (≈1435) | `#0f1419` | удалить |
| `.contacts` (≈874) | `#181c20` | удалить |
| `.footer` (≈1002) | `#111417` | удалить |
| `.rr` (≈3418) | свой градиент | удалить (`body` уже даёт тот же градиент); `.rr::before` (свечение) **оставить** |

### 3. `body.models-dark` (≈2641) — отдельная тёмная тема для `models.html`/`portfolio.html`

После унификации она избыточна. Делаем так:
- Удалить `body.models-dark { background: ... }` (фон уже из общего `body`).
- Удалить `body.models-dark .header*` правила (≈2646–2664) — пользователь явно сказал шапку не трогать.
- Оставить `body.models-dark .menu-overlay*` (≈2666–2672) — это стиль раскрытого меню, не «фон секции».

Класс `models-dark` остаётся в HTML — менять разметку не нужно.

## Что **не** трогаем

- `.header` (≈17) — остаётся светлой `#f5f5f5`.
- Любые компонентные фоны: карточки каталога, формы (`.service-form`, `.service-field input`), кнопки, модалки, оверлей меню, блоки «было/стало» в `.rr`.
- `.rr::before` — голубое свечение **только** в этой секции, как сейчас.
- Админка (`/admin/*`, `views/*.ejs`) — отдельный интерфейс.

## Возможные риски / визуальные эффекты

1. **`.why-us` теряет ярко-синюю подложку.** Сейчас это акцентная секция — после унификации станет такой же тёмной, как остальные. Декоративные радиальные «пятна» внутри (`.why-us-circle*`, ≈1145, ≈1157) дадут лёгкий синий тинт на тёмном — это нормально, в духе `.rr`.
2. **`.why-us-img-wrap` (≈1245, `rgba(255,255,255,0.08)`)** и подобные «стеклянные» панели на бывшем синем фоне теперь окажутся на тёмном — контраст немного снизится, читаемость не пострадает (текст белый).
3. **`background-attachment: fixed` на iOS Safari** иногда работает странно при скролле. Если визуально окажется плохо — fallback: `background-attachment: scroll` (градиент пойдёт во всю высоту страницы, на длинных страницах будет почти однородный — тоже нормально, цвета близкие).

## Файлы, которых меняем

- `css/style.css` — единственный.
- HTML / JS / шаблоны — без изменений.

## Проверка после внедрения

1. Открыть `http://localhost:3050/` (главная) — все секции на одном фоне, никаких видимых полос между ними. Шапка светло-серая.
2. `http://localhost:3050/#render-reality` — секция `.rr` визуально отличается от соседних только лёгким голубым свечением.
3. `http://localhost:3050/catalog`, `/portfolio`, `/models` — фон тот же, шапка светло-серая.
4. Пролистать каждую страницу до конца — фон не «съезжает» в другой оттенок.
5. Мобильный вид (DevTools, узкая ширина) — то же самое; на iOS особое внимание fixed-фону.
