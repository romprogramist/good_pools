## Архитектура

### 1. Страница политики

**Файл:** `privacy.html` (корень репо, рядом с `index.html`).

Структура (по образцу poolmann.ru/privacy):

1. Общие положения (Оператор, реквизиты, контакты)
2. Термины и определения
3. Принципы обработки ПД
4. Права и обязанности субъекта ПД и Оператора
5. Правовые основания обработки
6. Состав ПД, цели обработки, правовые основания
7. Порядок и условия обработки
8. Меры защиты ПД, порядок отзыва согласия
9. Cookies и аналитика
10. Трансграничная передача
11. Конфиденциальность
12. Изменение политики

Стиль страницы — минимальный, текстовый, наследует базовые стили сайта (header/footer + контейнер с типографикой). Без бокового меню. Якоря на разделы.

### 2. Футер

В футер всех 4 публичных страниц (`index.html`, `catalog.html`, `models.html`, `portfolio.html`) добавить ссылку:

```html
<a href="/privacy.html">Политика конфиденциальности</a>
```

### 3. Cookie-плашка

**Файлы:** `js/cookie-banner.js` (новый), `css/cookie-banner.css` (новый).

- Программно создаёт фиксированный блок снизу.
- При первом заходе показывается. После клика «Принять» — `localStorage.setItem('cookieAccepted', '1')` и блок удаляется.
- Подключается на все 4 публичные страницы + `privacy.html`.

### 4. Чекбоксы согласия в формах

Каждый из 5 модулей (`service`, `ask`, `consult`, `quiz`, `interest-popup`) добавляет в свою разметку:

```html
<label class="consent">
  <input type="checkbox" required name="consent">
  Я согласен с <a href="/privacy.html" target="_blank">Политикой обработки персональных данных</a>
</label>
<label class="consent consent--marketing">
  <input type="checkbox" name="marketing">
  Согласен получать рекламные сообщения и информацию об акциях
</label>
```

Кнопка submit блокируется JS, пока обязательный чекбокс не отмечен.

Стили чекбоксов — общий файл `css/consent.css`, подключается на всех страницах с формами.

### 5. Бэкенд `POST /api/leads`

**Файл:** `routes/api.js` (правка).

- Принимает JSON `{ source, name, phone, email?, payload?, consent: true, marketing: bool }`.
- Валидирует: `consent === true`, `name` непустое, `phone` нормализованный по маске.
- Пишет в `leads`:
  - `id`, `source`, `name`, `phone`, `email`, `payload` (JSONB — для квиза и т.п.), `consent_given=true`, `consent_marketing`, `consent_at=now()`, `consent_ip`, `policy_version`.
- Возвращает `{ ok: true }` или 400 с ошибкой.
- Без капчи и без rate-limit на этом шаге (отдельная задача).

### 6. Миграция БД

**Файл:** `db/migrations/0003_leads.sql`.

```sql
CREATE TABLE IF NOT EXISTS leads (
  id BIGSERIAL PRIMARY KEY,
  source VARCHAR(32) NOT NULL,
  name VARCHAR(255) NOT NULL,
  phone VARCHAR(32) NOT NULL,
  email VARCHAR(255),
  payload JSONB,
  consent_given BOOLEAN NOT NULL DEFAULT FALSE,
  consent_marketing BOOLEAN NOT NULL DEFAULT FALSE,
  consent_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  consent_ip VARCHAR(64),
  policy_version VARCHAR(32) NOT NULL,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX IF NOT EXISTS leads_created_at_idx ON leads (created_at DESC);
CREATE INDEX IF NOT EXISTS leads_source_idx ON leads (source);
```

`db/init.sql` обновляется тем же DDL, чтобы fresh install сошёлся.

### 7. Унификация фронта

В каждом из 5 модулей убирается `console.log` + TODO, добавляется:

```js
fetch('/api/leads', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ source: 'consult', name, phone, consent: true, marketing })
}).then(...).catch(...);
```

При ошибке сети — UI показывает «Не удалось отправить, попробуйте позже» (без попыток восстановления).
