# Замена квиза на прямую форму заявки

**Дата:** 2026-06-12
**Статус:** одобрено

## Проблема

При клике на «Рассчитать стоимость» (`[data-quiz-open]`) открывается многошаговый квиз
(размер → отделка → опции → бюджет → сроки → контакты). Нужно убрать вопросы и при клике
сразу показывать итоговую форму заявки (имя + телефон + согласие).

## Решение

Последний шаг квиза (`contacts`) уже и есть форма заявки. Убираем шаги-вопросы, оставляя
только контактный шаг. Вся обвязка модала переиспользуется. **Меняется только `js/quiz.js`.
HTML и CSS не трогаем.**

### Изменения в `js/quiz.js`

1. **Комментируем 5 шагов-вопросов** в массиве `STEPS` (`size`, `finish`, `options`,
   `budget`, `timing`) — остаётся только `contacts`. `TOTAL_STEPS = STEPS.length` станет = 1,
   поэтому:
   - модал открывается сразу на контактной форме;
   - кнопка «← НАЗАД» скрыта (`showBack = state.step > 1`);
   - кнопка действия — «ПОЛУЧИТЬ ПРЕДЛОЖЕНИЕ» (`isLast === true`).

2. **`validateStep`** — контактная проверка (имя/телефон) сейчас на `case 6`. Переносим её
   на шаг 1, проверки вопросов (`case 1..5`) комментируем.

3. **`handleSubmit` → payload** — поля `size/finish/options/budget/timing` комментируем,
   заявка уходит только с именем/телефоном. `source` остаётся `'quiz'`, эндпоинт
   `/api/leads` без изменений.

4. **`makeInitialState`** — поля ответов на вопросы (`size/finish/options/budget/timing`)
   комментируем, оставляем `name`/`phone`.

### Что сохраняется

Заголовок «РАССЧИТАТЬ СТОИМОСТЬ» + плашки про выгоду, маска телефона, чекбоксы
согласия/маркетинга (`ConsentHelper`), экран «Спасибо» (`showThankYou`), аналитика
(`ym reachGoal 'form_submitted_successfully'`).

### Стиль комментирования

Шаги и поля **закомментированы, не удалены**, с маркером-комментарием (напр.
`// QUIZ-DISABLED 2026-06-12: ...`), чтобы вернуть квиз можно было одной правкой.

## Охват

`js/quiz.js` подключён на `index.html`, `models.html`, `portfolio.html`, `catalog.html`.
Кнопки `[data-quiz-open]`: геройская на index + плавающие на всех 4 страницах. Так как файл
общий, правка автоматически действует везде — HTML не меняем.

## Кэш

Опционально бампнуть `?v=` у `<script src="js/quiz.js">` в 4 html-файлах, чтобы браузеры
подтянули новую версию. Для локального `--watch` достаточно hard-refresh.

## Вне охвата

- Форма консультации (`js/consult.js`) — не трогаем.
- Дизайн/вёрстка формы — без изменений.
- Серверная часть `/api/leads` — без изменений.
