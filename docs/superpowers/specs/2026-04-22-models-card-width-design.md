# Модели: шире карточки, фото целиком

**Дата:** 2026-04-22

## Цель

На `models.html` карточки каталога должны быть шире, а фото показываться целиком, без обрезания по краям. Сейчас `.mcard-img` квадратная с `object-fit: cover` — горизонтальные фото бассейнов срезаются сверху/снизу и по бокам.

## Решение — вариант B

Ровная сетка (все карточки одинаковые). Полосы-поля для фото не-16:10 сливаются с тёмным фоном `.mcard-img`, визуально почти незаметны.

## Изменения (только CSS, `css/style.css`)

1. `.models-grid` (около line 2726): `grid-template-columns: repeat(3, 1fr)` → `repeat(2, 1fr)`.
2. `.mcard-img` (около line 2749): `aspect-ratio: 1 / 1` → `aspect-ratio: 16 / 10` (как в `.pcard-img` портфолио — консистентно).
3. `.mcard-img img` (около line 2759): `object-fit: cover` → `object-fit: contain`.
4. В `@media (max-width: 1199px)` правило `grid-template-columns: repeat(2, 1fr)` становится избыточным (совпадает с дефолтом), но убирать его не обязательно — оставляем как есть, чтобы `gap: 24px` продолжал действовать.
5. Bump версии querystring в `models.html` для `css/style.css?v=…` — чтобы кеш у пользователей инвалидировался.

## Что НЕ трогаем

- JS (`js/models.js`, `js/gallery-modal.js`) — логика карточки и модалка не меняются.
- Badge и размерная плашка (`.mcard-badge`, `.mcard-size`) — позиционированы абсолютно относительно `.mcard-img`, остаются по углам контейнера, это ок (на letterbox-поле, не на фото — так даже читабельнее).
- Breakpoint `max-width: 767px` → 1 колонка — остаётся.

## Проверка

- `http://localhost:3050/models.html?category=composite&model=zen` — карточка ZEN шире, фото видно целиком, по бокам могут быть узкие тёмные поля.
- Модалка открывается по клику как раньше.
- На планшете (≤1199px) — 2 колонки, меньший gap.
- На мобильном (≤767px) — 1 колонка.
